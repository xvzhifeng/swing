应用页面结构方案

选择你认为最合理的页面组织方式

A

三页式（首页 + 运动 + 历史）

📱 首页
羽毛球运动
开始运动
查看历史
运动页
实时数据
控制按钮
历史页
统计图表
记录列表

优点

  • 结构清晰，入口明确
  • 每页职责单一

缺点

  • 需要多次点击跳转
B

双页式（运动 + 统计）

🏸 运动模式
• 实时数据显示
• 开始/暂停/结束按钮
• 进入即可开始
📊 统计分析
• 本次运动总结
• 历史记录
• 图表对比
左右滑动或按钮切换页面

优点

  • 快速切换运动/查看数据
  • 符合手表应用习惯

缺点

  • 统计页可能内容较多
C

单页流式（运动中 → 运动结束 → 历史）

状态1: 运动中
显示实时数据 + 控制按钮
状态2: 运动结束
自动展示本次统计 + "查看历史"按钮
状态3: 历史记录
列表 + 图表 + "开始新运动"按钮

优点

  • 符合使用流程
  • 减少导航层级

缺点

  • 运动中无法快速查看历史
